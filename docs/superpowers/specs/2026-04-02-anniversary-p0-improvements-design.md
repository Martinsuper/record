# 纪念日页面 P0 阶段改进设计

## 概述

本设计文档涵盖纪念日页面的两项 P0 级改进：
1. 补全重复方式选项
2. 应用内提醒通知

## 一、补全重复方式选项

### 需求背景

当前表单只展示"每年"选项，但数据结构支持 `none/year/month/week/day` 五种重复类型，用户无法选择其他方式。

### 改动范围

**文件**：`src/components/AnniversaryForm.vue`

### 设计方案

#### UI 展示
- 5个选项平铺展示：每年、每月、每周、每天、不重复
- 第一行显示4个常用选项（每年、每月、每周、每天）
- 第二行显示不重复选项
- 选中样式与现有"显示模式"选项一致：渐变背景 + 白色文字 + 加粗

#### 数据逻辑
- `repeatType` 默认值保持 `'year'`
- 点击选项直接更新 `repeatType` 值
- 编辑模式下正确回显已保存的值

#### 布局示意
```
┌─────────────────────────────────────┐
│ 重复方式                             │
├─────────────────────────────────────┤
│ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ │
│ │ 每年 │ │ 每月 │ │ 每周 │ │ 每天 │ │
│ └──────┘ └──────┘ └──────┘ └──────┘ │
│ ┌──────────┐                        │
│ │  不重复   │                        │
│ └──────────┘                        │
└─────────────────────────────────────┘
```

## 二、应用内提醒通知

### 需求背景

纪念日核心价值是"不遗忘"，但当前无主动提醒机制，用户需主动进入纪念日页面才能看到即将到来的事件。

### 改动范围

**新建文件**：
- `src/components/AnniversaryReminder.vue` — 提醒卡片组件

**修改文件**：
- `src/pages/index/index.vue` — 集成提醒组件
- `src/utils/anniversary.ts` — 新增 `getUpcomingAnniversaries` 函数
- `src/store/anniversary.ts` — 新增 `getUpcomingAnniversaries` getter（可选）

### 设计方案

#### 提醒触发逻辑
- 触发时机：首页 `onMounted` 时检查
- 提醒范围：今天 + 未来3天内到达的纪念日
- 排序规则：按剩余天数升序（今天在最前）

#### 提醒内容
- 纪念日名称
- 天数提示文案：
  - 今天："今天！"
  - 明天："明天"
  - 其他："还有 X 天"

#### UI 设计
- **位置**：首页 header 下方，filter-section 上方
- **样式**：玻璃卡片风格（glass-card），与现有设计一致
- **高度**：根据提醒数量动态调整，最多显示3条
- **交互**：
  - 点击卡片任意位置跳转到纪念日页面
  - 右上角关闭按钮（×），关闭后本次会话不再显示
  - 使用 sessionStorage 或组件内部状态控制关闭状态

#### 无提醒时
- 不显示提醒卡片（组件不渲染）

#### 布局示意
```
┌─────────────────────────────────────┐
│ 📅 纪念日提醒                    ✕  │
├─────────────────────────────────────┤
│ • 恋爱纪念日 — 还有2天              │
│ • 妈妈生日 — 今天！                 │
└─────────────────────────────────────┘
```

```typescript
interface UpcomingAnniversary {
  id: string
  name: string
  days: number        // 剩余天数（0=今天，负数=已过）
  displayText: string // 显示文本

/**
 * 获取即将到来的纪念日
 * @param anniversaries 纪念日列表
 * @param daysRange 范围天数（默认3）
 * @returns 筛选后的列表，按天数排序
 */
function getUpcomingAnniversaries(
  anniversaries: AnniversaryData[],
  daysRange: number = 3
): UpcomingAnniversary[]
```

## 实现要点

### 重复方式计算逻辑更新

当前 `calculateAnniversary` 函数已支持各 repeatType，但需验证：
- 每月：计算到本月/下月同一天
- 每周：计算到本周/下周同一天
- 每天：总是显示"今天"或"明天"
- 不重复：已过则转为正计时显示

### 提醒数据来源

首页需访问 anniversary store：
- 导入 `useAnniversaryStore`
- 调用 `loadFromStorage()` 加载数据
- 使用 getter 或计算属性获取即将到来的纪念日

## 测试要点

1. **重复方式**
   - 切换各选项后保存，验证数据正确存储
   - 编辑模式下正确回显已保存选项
   - 各重复类型的计算逻辑正确显示天数

2. **提醒通知**
   - 首页正确显示今天+未来3天的纪念日
   - 无符合条件的纪念日时不显示卡片
   - 点击跳转正确进入纪念日页面
   - 关闭后本次会话不再显示
   - 提醒文案正确（今天/明天/还有X天）
