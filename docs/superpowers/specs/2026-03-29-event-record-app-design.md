# 事件记录应用设计规格说明书

**日期**: 2026-03-29
**项目**: 事件记录 UniApp 应用

---

## 1. 项目概述

### 1.1 目标

构建一个简洁美观的纯前端事件记录应用，支持 H5 和小程序双端运行，让用户能够快速记录事件、时间和事件类型。

### 1.2 核心特性

- **极简字段**: 事件名称 + 时间 + 类型（三个核心字段）
- **完全自定义类型**: 用户从零开始创建所有事件类型
- **本地存储**: 数据保存在用户设备，使用 uni.storage
- **清爽简洁风格**: 白色为主，少量点缀色，扁平化设计

### 1.3 技术栈

| 层面 | 技术选择 |
|------|----------|
| 前端框架 | UniApp (Vue3) |
| UI框架 | uView UI 2.x (u-view-plus) |
| 状态管理 | Pinia |
| 样式方案 | SCSS |
| 数据持久化 | uni.setStorageSync |

### 2.2 设计原则

- **数据与视图分离**: Pinia 管理状态，组件只负责渲染和交互

## 3. 数据模型

### 3.1 事件 (Event)

```javascript
{
  id: string,          // 唯一标识（时间戳生成）
  name: string,        // 事件名称
  typeId: string,      // 关联的事件类型ID
  time: number,        // 事件时间（Unix毫秒）
  createdAt: number    // 创建时间
}
```

### 3.2 事件类型 (EventType)

```javascript
{
  id: string,          // 唯一标识
  name: string,        // 类型名称
  color: string,       // 显示颜色（用于标签/标记）
  createdAt: number    // 创建时间
}
```

### 3.3 本地存储键

| 存储键 | 说明 |
|--------|------|
| `events` | 事件列表数组 |
| `eventTypes` | 事件类型数组 |

## 5. 核心交互流程

### 5.1 添加事件流程

```
用户点击添加按钮
    ↓
弹出添加事件浮层 (EventForm.vue)
    ↓
输入事件名称 → 选择类型 → 选择时间
    ↓
点击保存
    ↓
数据写入 Pinia store → 同步到 uni.storage
    ↓
浮层关闭 → 列表刷新显示新事件
```

### 5.2 新建类型流程

```
在添加浮层点击"新建类型"
    ↓
弹出新建类型子浮层 (TypePicker.vue 内嵌)
    ↓
输入类型名称 → 选择颜色
    ↓
点击保存
    ↓
类型写入 store → 同步到 storage
    ↓
自动选中新类型 → 返回添加浮层
```

### 5.3 筛选事件流程

```
在筛选栏选择类型/时间范围
    ↓
更新筛选条件 (Pinia state)
    ↓
EventList 组件重新计算显示列表
    ↓
界面实时更新
```

### 5.4 删除事件流程

```
事件卡片左滑
    ↓
显示删除按钮
    ↓
点击删除
    ↓
弹出确认提示 (uView modal)
    ↓
确认后从 store 移除 → 同步 storage
    ↓
列表刷新
```

## 6. 状态管理设计

### 6.1 Event Store (store/event.js)

```javascript
// Pinia store 定义
export const useEventStore = defineStore('event', {
  state: () => ({
    events: [],           // 事件列表
    filterType: null,     // 筛选类型ID (null=全部)
    filterTimeRange: 'all' // 筛选时间范围
  }),

  getters: {
    // 获取筛选后的事件列表
    filteredEvents: (state) => {
      let result = state.events
      // 按类型筛选
      if (state.filterType) {
        result = result.filter(e => e.typeId === state.filterType)
      }
      // 按时间筛选
      result = filterByTimeRange(result, state.filterTimeRange)
      // 按时间倒序排列
      return result.sort((a, b) => b.time - a.time)
    },

    // 统计数据
    totalCount: (state) => state.events.length,
    monthCount: (state) => /* 本月数量计算 */
  },

  actions: {
    // 添加事件
    addEvent(event) { ... },
    // 删除事件
    deleteEvent(id) { ... },
    // 从 storage 加载
    loadFromStorage() { ... },
    // 保存到 storage
    saveToStorage() { ... }
  }
})
```

## 7. 核心组件说明

### 7.1 EventList.vue

**职责**: 显示事件卡片列表，支持左滑删除

**依赖组件**:
- `u-card`: uView 卡片组件
- `u-swipe-action`: uView 滑动操作组件

**Props**: 无，直接从 store 读取 filteredEvents

**Events**:
- `@delete`: 删除事件时触发

### 7.2 EventForm.vue

**职责**: 添加事件浮层表单

**依赖组件**:
- `u-popup`: uView 弹出层
- `u-input`: 输入框
- `u-picker`: 类型选择器
- `u-datetime-picker`: 时间选择器

**Props**:
- `visible`: Boolean, 控制显示

**Events**:
- `@save`: 保存事件
- `@close`: 关闭浮层

### 7.3 TypePicker.vue

**职责**: 类型选择 + 新建类型功能

**依赖组件**:
- `u-picker`: 选择器
- 内嵌新建类型表单

### 7.4 FilterBar.vue

**职责**: 顶部筛选器栏

**依赖组件**:
- `u-dropdown`: uView 下拉菜单
