# 事件列表页左滑改长按设计方案

## 背景

当前事件列表使用左滑（swipe）手势触发编辑/删除菜单，存在滑动过程中误触发的用户痛点。用户反馈在快速滑动列表时，偶尔会不小心展开菜单，影响体验。

## 目标

将触发方式从"左滑"改为"长按"，减少误触发，提升交互可靠性。

## 详细设计

### 1. 触发机制

| 参数 | 值 | 说明 |
|------|-----|------|
| 长按时长 | 800ms | 用户需长按至少 800ms 才触发菜单 |
| 触发区域 | 整个事件卡片 | 覆盖整个卡片可触发 |
| 取消条件 | 松开手指或滑动距离 > 20px | 区分长按与滑动 |

### 2. 气泡菜单 UI

```
┌────────────────────────────────────┐
│  ✏️ 编辑              🗑️ 删除     │
└────────────────────────────────────┘
```

- **位置**：在长按位置上方弹出（向上偏移 10px），避免遮挡操作区域
- **尺寸**：宽度约 200px，高度约 50px
- **背景**：白色 (#FFFFFF)，圆角 12px
- **阴影**：`0 4px 12px rgba(0, 0, 0, 0.15)`
- **选项**：
  - 编辑：蓝色 (#3B82F6)，带图标 ✏️
  - 删除：红色 (#EF4444)，带图标 🗑️
- **布局**：Flex 布局，两端对齐，选项间有间距

### 3. 交互流程

```
用户点击卡片
    ↓
开始计时 (800ms)
    ↓
┌────────────────────────────────────┐
│  用户在 800ms 内松开                │  用户保持长按 ≥ 800ms
│  → 取消，不触发任何操作             │  → 气泡菜单出现
│  (视为普通点击，后续可进入详情页)    │  ↓
└────────────────────────────────────┘                    点击菜单选项
    ↓                                              ↓ 执行编辑/删除
    ↓                                    气泡菜单隐藏
    ↓
用户点击选项后执行对应操作
```

### 4. 状态管理

- **isLongPressing**：是否正在长按
- **menuVisible**：气泡菜单是否可见
- **menuPosition**：气泡菜单位置 { x, y }

### 5. 边界情况

| 场景 | 处理 |
|------|------|
| 菜单超出屏幕边界 | 自动调整位置，确保不超出视口 |
| 快速连续长按 | 上一个菜单关闭后再响应新的长按 |
| 长按后点击菜单外区域 | 菜单隐藏，不执行任何操作 |
| 长按后移动手指 | 取消长按，不触发菜单 |

## 技术实现建议

### 方案 A：自定义实现

使用 uView 的 LongPress 事件或原生事件：

```vue
<u-cell @longpress="showMenu">
```

- 优点：可控性高
- 缺点：需要自己实现气泡菜单定位和隐藏逻辑

### 方案 B：使用 uView Popup/Popover

利用 uView 的 `u-popover` 组件：

- 优点：已有定位和动画逻辑
- 缺点：可能需要调整样式

## 验收标准

1. 长按 800ms 后菜单出现，短按（< 800ms）不触发
2. 菜单位置在长按位置上方，不超出屏幕
3. 点击编辑/删除选项可正常执行对应操作
4. 点击菜单外部区域可关闭菜单
5. 列表正常滚动不受影响
